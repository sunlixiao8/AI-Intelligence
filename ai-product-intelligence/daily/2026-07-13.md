### 1. Waifu

#### 产品
- **名称**：Waifu
- **URL**：https://trustmrr.com/startup/waifu
- **类别**：AI / 娱乐 / 图像生成
- **目标用户**：二次元爱好者、想要个性化虚拟伴侣/头像的用户。

#### 用户问题
- **解决什么问题**：让用户能快速生成自己专属的、风格一致的二次元“角色”（Waifu）。传统手绘动漫角色门槛高、耗时长，而且很难获得一张风格统一、表情各异的个人角色图。
- **为什么付费**：这不是一次性好奇心。用户需要大量不同姿态、表情、场景的角色图用于社交媒体头像、聊天室人物卡、甚至个人创作。这是一种“自我表达”和“身份认同”的高频需求。付费意愿高，因为他们想要“专属感”和“高质量”。

#### 商业信号
- **定价模式**：未获取具体定价，但这类产品通常采用积分制、订阅制（不同生成数量和质量的分层）或一次性购买角色包。
- **月收入估算**：**$3,900/月**
- **置信度**：**高**
- **证据**：TrustMRR明确列出该产品当前月收入为$3,900（MRR），要价$80k。这是可靠的官方市场数据。

- **增长或牵引力证据**：
    - 其在TrustMRR上挂牌出售，本身就是一种牵引力信号，表明它是一个有稳定现金流的可交易资产。
    - 基于AI的二次元图像生成市场非常庞大且活跃，这意味着有潜在客户基础。

#### 为什么可能赚钱
- **使用频率**：高。用户会频繁地生成不同主题（节日、心情、活动）的角色图像。
- **紧迫程度**：中高。对于在动漫社群、虚拟主播、社交媒体上活跃的用户，拥有一套高质量的个人角色形象是刚需。
- **预算负责人**：用户个人（娱乐预算）。
- **切换成本**：中等。一旦用户在一个平台投入时间调教模型、生成大量图片，迁移到另一个平台的学习成本和情感成本会增加。但开源和竞品众多，优势不绝对。

#### 产品逻辑与技术路径
- **适合初学者的 MVP 范围**：一个垂直的、针对个人角色定制的图像生成工具。可以只支持生成特定风格的“头像”和“半身像”，无需支持全身或复杂场景。
- **核心工作流**：
    1. 用户上传3-5张参考图（或选择平台预设风格）。
    2. 用户选择角色基础特征（发型、眼睛颜色等）。
    3. 用户输入“角色名 + 动作/场景”提示词。
    4. AI模型生成图像，用户可以选择“最喜欢的一张”作为角色锚点。
    5. 之后所有新生成，都基于这个锚点保持人物一致性。
- **需要的数据或输入**：用户上传的角色图像、用户输入的文本提示词。
- **推荐技术栈**：
    - **核心AI**：Stable Diffusion + LoRA微调技术。这是目前实现“角色一致性”最成熟、最便宜的路径。可以使用Replicate或Fal.ai的托管API快速上手。
    - **前端**：Next.js + Tailwind CSS。
    - **后端**：Next.js API Routes。
    - **数据库/认证**：Supabase（存储用户账户、生成的图片URL和角色元数据）。
    - **支付**：Stripe或Lemon Squeezy（对AI、图像类平台友好）。
- **AI模型用法**：图像生成（Stable Diffusion）、文本到图像（用户Prompt）。
- **最大难点**：**角色一致性**。让AI生成同一个人在不同动作、表情、角度下都“长得一样”是最大的技术挑战，也是产品核心价值。用户会因为“长得不像”而放弃。
- **降低难度的捷径**：
    1. **不做即时一致性**。MVP可以只提供“风格一致”，即保持画风、色调但允许角色特征略有差异。
    2. **使用Civitai等社区的开源LoRA模型**。直接使用别人训练好的、擅长某种画风的LoRA作为起点，专注做产品体验和UI。
    3. **聚焦特定动漫**。只做《原神》、《崩坏：星穹铁道》等特定IP的角色风格，可极大降低模型训练难度。

#### 适合 OPC 初学者的机会
- **可以切入的更小众版本**：
    - **“我的虚拟主播形象生成器”**：专为Vtuber提供半身、全身、表情差分的高质量角色卡。
    - **“DnD角色肖像生成器”**：针对桌游玩家，生成奇幻风格的角色画像。
- **第一批应该访谈的客户群体**：
    1. 在Discord/NGA动漫频道活跃的用户。
    2. 个人画师约稿群里的甲方（询问他们为一张角色图愿意付多少钱）。
- **1周验证实验**：
    1. 在C站（Civitai）找一个免费高好评的动漫LoRA。
    2. 用Gradio或Streamlit搭一个极简页面，让用户上传一张角色参考图，输入提示词，调用Replicate API生成。可以预置5个Prompt（如“微笑”、“生气”、“跑步”）。
    3. 在动漫社群发帖：“给站长看看你家的Waifu，免费生成5张图！”看私信和帖子的反馈量。

- **月收入估算**：**$6,200/月**
- **置信度**：**高**
- **证据**：TrustMRR明确列出其月收入为$6,200，要价$440k。这是一个高价值的收入资产。

- **增长或牵引力证据**：
    - **高MRR**：$6.2k月收入对于一个单一功能的移动应用来说非常出色。

#### 为什么可能赚钱
- **使用频率**：应用中高（每月设置一次，但提醒功能每周触发）。学习曲线极低。
- **紧迫程度**：极高。这是一个刚需，忘记的后果严重。
- **预算负责人**：用户个人（健康管理预算，单价低但决策快）。
- **切换成本**：极低。但用户粘性极高，它会变成一种习惯。只要App不崩溃，用户几乎不会主动卸载。

#### 产品逻辑与技术路径
- **适合初学者的 MVP 范围**：一个基于日历的、带推送通知的App。不需要任何健康知识图谱或复杂的分析。核心就是计算和提醒。

    1. 用户注册，选择“避孕环”类型（或手动输入周期）。
    2. 用户设定“放置日期”或“取出日期”。
    3. App自动计算并设置一个28天的循环周期（21天放置 + 7天取出）。
    4. App在关键时间点（即将到期、已到期）发送推送通知。

    - **平台**：React Native (Expo) 或 Flutter。一次性覆盖iOS和Android。
    - **后端/API**：Supabase（用于存储用户设定和通知令牌）。
    - **通知服务**：Expo Push Notifications 或 Firebase Cloud Messaging（FCM）。
    - **AI模型用法**：**本例不需要AI**。这是一个纯粹的产品和运营驱动型产品。它证明了在特定领域，好的产品逻辑和解决实际问题比AI更重要。
- **最大难点**：**技术实现几乎无难点**。最大挑战在于**应用商店审核**（健康类App）和**获客**（如何让目标用户发现你）。
- **降低难度的捷径**：自己充当第一批用户，用手表、便签、日历App就能做出原型。但代码层面，用Expo和Supabase可以非常快地写完核心逻辑。**完全不用碰AI**。

    - **“隐形眼镜更换提醒器”**：记录日抛、月抛、年抛时间。Mrr可能类似。

- **降低难度的捷径**：
    - **功能**：不要做“通用Claude工具”，而是 **“Claude + PDF分析器”**。
    - **UI/UX**：专注于一个极端简单且优雅的界面。例如，一个能通过拖拽PDF文件进浏览器窗口，然后直接开始多轮对话、分析合同的工具。
    - **营销**：在Product Hunt发布一个“最小可用但解决具体问题”的版本。比如，专为分析“500页法律合同”而设计。

#### 适合 OPC 初学者的机会
- **可以切入的更小众版本**：
    - **“Claude Prompt 模板市场”**：一个让用户共享、评分、销售好用的Claude Prompt的平台。
    - **“Claude 项目协作平台”**：让团队能共享上传给Claude的文档知识库，并进行协作。
- **第一批应该访谈的客户群体**：
    1. 在X/Twitter上分享Claude使用技巧的开发者。
    2. 在技术社区抱怨Claude集成困难的开发者。
- **1周验证实验**：
    1. 写一篇博客《手把手教你用Supabase + Claude 5分钟搭建一个文档问答机器人》。
    2. 将源码上传到GitHub。
    3. 在结尾留个“觉得有用？我们正在开发一个更强大的无代码版本，点击Waitlist”，看有多少开发者愿意留下邮箱。

## 今日模式

**“用极致具体的细分需求，取代通用功能”**

- **反例**：做一个“通用的AI客服聊天机器人”。
- **正例**：Contraceptive Ring Reminder只为一个设备提供提醒。Waifu只做二次元角色图。GetMentioned只做PR Outreach。商业模式取得成功的，往往是那些把事情做窄、做透的产品，而不是大而全的平台。小产品占据一个明确的用户心智，护城河来自“解决这个具体问题上，它最好”。

## 建造雷达

1.  **<我的药盒提醒器>**
    - **产品**：适合老年人或慢性病患者的用药提醒App。无需AI，核心是图片识别（用户拍一下药盒，AI自动识别药品名和服用规则） + 多维提醒（App、微信、甚至智能硬件）。**核心差距**：Contraceptive Ring Reminder是$6.2k MRR的产品，用药提醒的MRR天花板可能更高。

2.  **<AI 代码审查文案生成器>**
    - **产品**：针对Git Pull Request的AI助手。不是审查代码逻辑，而是**自动为每一行改动生成“优美的、符合Commit规范的英文/中文注释和描述”**。开发者没有动力写好的commit message，但团队又需要。**核心差距**：极低成本解决高频烦恼，付费者可能是项目经理或技术主管。

3.  **<小团队的一站式PR Launch Kit>**
    - **产品**：针对月收入$1k-$5k的SaaS小团队。当你有新功能发布时，它自动：
        - 从你的更新日志中提取关键信息。
        - 用AI生成针对Product Hunt、Twitter、LinkedIn的发布帖子和表格。
        - 生成3篇不同风格的新闻稿草稿（TechCrunch风、行业垂直媒体风、用户故事风）。
        - **核心差距**：GetMentioned ($793 MRR)证明了小公司愿意为PR效率付费。这个版本更聚焦于“发布”这个场景。

## 用户访谈问题

1.  **关于现有解决方案：** “在遇到[问题X]时，你目前是怎么处理的？是用什么工具、什么流程？能告诉我整个具体操作步骤吗？”（寻找痛点：现在的方法有没有哪里让你不爽？）
2.  **关于付费意愿：** “如果有一个工具/App能解决这个令人不爽的地方，你愿意每月支付多少钱？或者是一次性支付多少钱？”（直接探底价格，确认是否真的有预算。）
3.  **关于优先级：** “除了这个问题，目前工作中还有没有什么其他让你更头疼、更想解决的事情？”（了解核心用户画像下，是否存在更需要解决、更有可能付费的其他问题。）
