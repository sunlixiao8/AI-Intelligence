# 每日 AI 产品商业情报

## 目标

帮助刚开始学习 AI 产品开发的个人创业者，通过研究真实且看起来能赚钱的产品，培养商业嗅觉。重点关注高频需求、明确付费意愿、具体使用场景，以及能反哺后续选题判断的经验。

目标阅读时长：约 20 分钟。简报要有足够深度，让读者能学习其他产品和创业者的经验，但不要写成冗长的市场报告。

## 产品筛选

每期选择 3-4 个产品。优先选择 AI 产品；如果某个非 AI SaaS、插件或工作流工具能体现有价值的商业模式，也可以纳入。

优先选择至少符合以下两项的产品：

- 解决反复出现的工作或生活问题，而不是一次性的好奇需求。
- 有可见的定价、订阅、付费积分、企业方案、市场销售记录，或公开收入/MRR 线索。
- 服务具体角色、工作流或行业，而不是模糊的大方向。
- 有增长或牵引力信号：评论、流量、发布讨论、搜索热度、融资、社会证明、应用排名、Product Hunt 活动、市场安装量、创始人公开更新等。
- 初学者借助 AI，有可能做出一个更小的 MVP 版本。

避免选择只有模糊 AI 包装、纯演示性质、巨型平台型机会，或完全无法推断收入的产品。

## 信息来源优先级

优先从以下来源开始，必要时再扩展搜索：

- https://trustmrr.com/
- https://www.toolify.ai/zh/
- https://theresanaiforthat.com/
- Product Hunt、Indie Hackers、X/Twitter 创始人帖子、应用商店、Chrome Web Store、Shopify App Store、G2/Capterra、Similarweb 类流量估算、产品定价页、更新日志、公开访谈。

当收入没有官方公开时，保守估算，并标注置信度。

## 收入估算

每个产品需要提供以下三类之一：

- 官方收入：来自创始人、公司、市场平台或可信数据库的公开 MRR/ARR/收入。
- 推测收入：根据定价、流量、评论数、公开用户数、安装量、市场排名等线索估算。
- 未知：说明缺少哪些关键信号。

使用以下格式：

- 月收入估算：`$X-$Y/月`
- 置信度：`高`、`中` 或 `低`
- 证据：简短解释并附链接。

不要把低置信度收入估算写成确定事实。

## 每日简报输出

使用中文简体输出。内容要实用、清晰、适合快速浏览。

每个产品按以下结构输出：

1. **产品**
   - 名称、URL、类别、目标用户。
2. **用户问题**
   - 它解决了什么痛苦、高频、具体的问题。
   - 用户为什么可能付费，而不只是好奇试用。
3. **商业信号**
   - 定价模式。
   - 月收入估算与置信度。
   - 增长或牵引力证据。
4. **为什么可能赚钱**
   - 使用频率。
   - 紧迫程度。
   - 预算负责人。
   - 切换成本或使用习惯优势。
5. **产品逻辑与技术路径**
   - 适合初学者的 MVP 范围。
   - 核心工作流。
   - 需要的数据或输入。
   - 推荐技术栈。
   - AI 模型用法：分类、生成、抽取、检索、智能体工作流、视觉、语音或自动化。
   - 如果“我”来开发，会遇到的最大难点。
   - 可降低难度的捷径或更简单的第一版。
6. **适合 OPC 初学者的机会**
   - 可以切入的更小众版本。
   - 第一批应该访谈的客户群体。
   - 1 周验证实验。

结尾包含：

- **今日模式**：一个可复用的商业洞察。
- **建造雷达**：2-3 个受今日产品启发的小产品想法。
- **用户访谈问题**：3 个可以直接问潜在用户的问题。

## 推荐初学者技术栈

除非产品明显需要其他方案，否则优先使用简单、常见的工具：

- 前端：Next.js、React、Tailwind CSS。
- 后端：Next.js API Routes、FastAPI 或 Supabase Edge Functions。
- 数据库/认证：Supabase。
- 支付：Stripe 或 Lemon Squeezy。
- AI：OpenAI API 或同类模型服务。
- 检索：pgvector/Supabase Vector 或托管向量数据库。
- 自动化：定时任务、队列，或 Zapier/Make 类集成。
- 数据分析：PostHog、Plausible，或简单事件表。

## 质量标准

- 包含信息来源链接。
- 引用产品或收入信号时尽量写明日期。
- 明确区分事实与估算。
- 优先写具体工作流，不写泛泛的 AI 描述。
- 突出初学者真实可以做出来的版本。
- 如果信息不确定，说明下一步应该验证什么。
- 内容权重保持在约 70% 商业判断、30% 产品/技术实现。
